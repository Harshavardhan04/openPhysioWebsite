Testing - Overview

Our development process was underpinned by a comprehensive testing strategy, ensuring both the backend and frontend components of our application met stringent quality standards. OpenPhysio is a data transferring and storing medical technology, therefore it is necessary to produce thorough tests to ensure itâ€™s usability, accuracy and effectiveness matches our moSCoW requirements.

Test Scope
  1. FHIR is the data transferral standard commonly used by health practitioners. As the project relies heavily on transferring medical data then it is crucial that the ability of the software to transfer data- in line with these standards- should be tested. This ensures that the data is both transferred correctly but also in line with the industry standard.
  2. As it is medical practitioners who will be the primary users of the software, testing that the user experience is comfortable and the interface is similar to healthcare norms is vital to the sleek integration of the project into service.
  3. The aim of the project is to provide a succinct yet comprehensive view of patient performance. Therefore, testing the efficacy of its ability to provide the user with a visualization of patient improvement over time must be conducted.

In the backend, we adopted a Test-Driven Development methodology, which played a pivotal role in shaping our core functionality. This approach involved writing detailed unit tests prior to developing functional code, establishing a robust foundation for our backend services. Through this rigorous process, we were able to identify and address potential issues early in the development cycle, enhancing code reliability and maintainability. Test-Driven Development also benefitted the development through allowing the developer to consider the purpose of each unit in the wider context of its subsystem. This also allowed for the precise consideration of the exact specifications of each function before the development process. The team continually adapted the backend to pass all unit tests and extending the unit tests when more functionality was required.

For the frontend, our strategy encompassed an exhaustive User Acceptance Testing regimen, aimed at ensuring the user interface aligned perfectly with end-user expectations and usability standards. An integral part of our frontend testing was dedicated to responsive design testing, where we meticulously tested the application across a spectrum of devices and screen sizes. This responsive design testing was instrumental in confirming that our application adapted gracefully to different viewing contexts, providing an optimal user experience regardless of device or browser.

The user acceptance testing process for the frontend not only assessed functional correctness and user interaction but also placed a strong emphasis on the application's aesthetic adaptability and responsiveness. By rigorously testing and fine-tuning the UI/UX across diverse environments, we ensured our application was accessible, intuitive, and engaging for physiotherapists, and in line with NHS standards.

Following the successful validation of individual backend and frontend components, we moved to the integration phase, where these components were synergized to function as a cohesive unit. Once they had been integrated, they needed to be tested. It made the most sense to test these manually as synthetic data had been generated and the functionality of the site was predetermined- therefore it was a matter of testing whether the frontend could accurately display the stored data and transfer input to the backend for processing and storage.

The team is very happy with the feedback and, through user acceptance, it has become clear that there is enthusiasm towards deploying the software to production. All of the teams internal tests were passed and in good time, partly due to large efforts during coding sprints, and the user acceptance tests generated feedback which proved to be invaluable.

User Acceptance Testing

User Acceptance Testing played an integral role in the frontend development of our healthcare application. This phase was engineered to validate the application against real-world scenarios and user requirements, ensuring that every UI component resonated with the practical needs of healthcare professionals.

During user acceptance testing, we engaged with a diverse group of physiotherapists, to interact with the application in a staged environment that simulated their daily operational settings. This process involved executing predefined test cases that covered a broad spectrum of functionalities, including patient data entry, retrieval of past consultation details, and the generation and export of reports.

Key Elements of User Acceptance Testing included:
  • Functionality: Each function, like entering consultation numbers or selecting SNOMED codes, was tested to verify that it performed as expected. Actions such as navigating back to the dashboard, adding and removing SNOMED codes, and downloading full data were thoroughly tested to confirm operational integrity.
  • Usability: The ease with which users could interact with the UI elements, such as buttons, dropdowns, and text inputs, was assessed. The goal was to ensure that users could accomplish their tasks with intuitiveness and efficiency.
  • Data Accuracy: The accuracy of data displayed in the consultation tables and the corresponding values in the trend charts was critically examined. Users verified that the observed and desired values reflected correctly and that the data visualization accurately represented their historical trends.
  • Error Handling: The application's behavior in response to incorrect inputs or actions was also tested. This included entering non-numeric values for consultation numbers, SNOMED codes, or leaving mandatory fields like the 'Problem Description' empty.

Feedback from User Acceptance Testing was invaluable, leading to several iterative enhancements that optimized the workflow and user experience. This ensured that upon release, the application was not only functionally robust but also aligned with the high usability standards expected by healthcare professionals.

Responsive Design Testing

Responsive Design Testing was critical in ensuring our application's interface was optimized for various devices, screen sizes, and orientations. Given the diverse nature of device usage in healthcare settings, from desktop computers to tablets, our application needed to be highly flexible and adaptive.

The testing process included:

  • Layout fluidity: We tested the layout adjustments across different screen sizes to ensure that elements like the consultation data table and trend chart resized and repositioned correctly. Special attention was given to the consultation data table, ensuring that columns adjusted without compromising readability or functionality.
  • Touch Interactions: On touch devices, we focused on elements like dropdowns and buttons to ensure they were easily selectable and provided the necessary feedback to touch interactions.
  • Cross-Browser Compatibility: The application was tested on multiple browsers to ensure consistent performance and appearance. We also ensured that the visual elements like the 'BACK TO DASHBOARD' button and 'DOWNLOAD FULL DATA' buttons rendered correctly and maintained their legibility and accessibility.
  • Performance: We analyzed the application's performance, particularly the rendering of the trend charts, to ensure smooth and quick loading times across all devices, with no compromise on the interactive features such as tooltips on hover.
  • Orientation Change: The application's adaptability to orientation changes on tablets and mobile phones was verified to ensure a seamless transition between landscape and portrait modes.
